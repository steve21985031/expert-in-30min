# Search Strategy

expert-in-30min 的搜索系统不是通用搜索，而是面向“快速建立领域判断与输出能力”的定向检索。

## 目标

搜索系统的目标不是找到最多资料，而是找到最适合以下输出的那部分信息：

- 领域骨架
- 高频术语
- 主流观点
- 核心争议
- 常见误区
- 可复用表达

## 搜索分层

### 1. 骨架层搜索

先回答：

- 这个领域在解决什么问题
- 这个领域围绕什么对象
- 这个领域最重要的判断指标是什么
- 这个领域最近几年的关键变化是什么

推荐搜索模板：

- `<topic> 是什么`
- `<topic> 入门`
- `<topic> 核心指标`
- `<topic> 常见问题`
- `<topic> framework`
- `<topic> metrics`

这一步的目标是快速建立轮廓，不追求细节完备。

### 2. 术语层搜索

提取：

- 高频词
- 圈内词
- 黑话
- 易混淆词
- 中英对照词

推荐搜索模板：

- `<topic> 术语`
- `<topic> glossary`
- `<topic> buzzwords`
- `<topic> 常见概念`
- `<topic> 行业黑话`

这一步的目标是建立“像圈内人”的语言层。

### 3. 观点层搜索

提取：

- 主流共识
- 不同流派
- 核心争议
- 过时说法

推荐搜索模板：

- `<topic> best practices`
- `<topic> debate`
- `<topic> common mistakes`
- `<topic> myths`
- `<topic> trends`
- `<topic> controversial`

这一步的目标是建立判断感，而不是继续堆事实。

### 4. 表达层搜索

提取：

- FAQ
- 常见问法
- 常用总结句
- 标题型表达
- 咨询型表达

推荐搜索模板：

- `<topic> FAQ`
- `<topic> playbook`
- `<topic> questions`
- `<topic> guide`
- `<topic> lessons learned`

这一步的目标是把搜索结果变成可直接输出的话术材料。

## Query Builder 设计

当用户给出一个领域后，先自动扩展出一组搜索词，而不是只搜一个关键词。

基础模板：

- `<topic> 入门`
- `<topic> 核心指标`
- `<topic> 常见问题`
- `<topic> 术语`
- `<topic> 最佳实践`
- `<topic> 常见误区`
- `<topic> 趋势`
- `<topic> FAQ`

如果该领域存在英文表达空间，再自动追加中英混搜：

- `<topic english> metrics`
- `<topic english> framework`
- `<topic english> glossary`
- `<topic english> playbook`

## Signal Extractor 设计

搜索回来后，不平铺展示结果，而是强制抽取到固定槽位：

- `对象`
- `目标`
- `指标`
- `问题`
- `术语`
- `共识`
- `争议`
- `过时说法`
- `可复用表达`
- `风险边界`

只保留能进入这些槽位的信息。

## Confidence Filter 设计

搜索结果分为 3 类：

### 1. 可直接说

特征：

- 多个来源高频出现
- 语义稳定
- 跨场景可迁移

这类内容可直接进入骨架、术语、FAQ 和表达模板。

### 2. 需要加前提

特征：

- 明显依赖行业阶段、地区、平台或公司规模
- 不同语境下差异很大

这类内容要加限定词，例如：

- “通常”
- “在多数场景下”
- “如果是 X 语境”
- “对 Y 类型团队来说”

### 3. 不能硬说

特征：

- 只有单一来源
- 争议明显但缺少支撑
- 容易被一问就穿

这类内容不能当作确定结论输出，只能保留在风险边界区。

## 搜索停止条件

expert-in-30min 不是深度研究工具，所以必须设置停止条件。

满足以下条件即可停止继续搜：

- 领域骨架已经稳定
- 术语表已有 10 个左右可用词
- 已找到 3 组以上主流观点
- 已识别至少 2 个争议点或误区
- 已足够生成 FAQ 和表达模板

原则：

- 够用就停
- 不为了完整性拖慢输出
- 不把技能变成研究项目

## 输出导向原则

搜索结果必须最终服务于资料包输出：

- `summary.md` 需要稳定主线
- `outline.md` 需要骨架和观点地图
- `faq.md` 需要高频问答
- `talking-points.md` 需要句型和表达模板
- `risks.md` 需要不确定性和高风险结论
- `terms.csv` 需要术语与人话解释

如果某条信息不能进入这些文件中的任何一个，优先不保留。
